# 🌸 Haru SMI – Prototype / v3.0 (Supabase Edition)

**Haru SMI** は更年期世代の女性に向けた「日々のコンディション記録 × AIアドバイス × コミュニティ」アプリです。
本リポジトリは、**Supabaseをバックエンドに採用し、実運用を見据えたデータ構造を持つ最新プロトタイプ**です。

---

## 📌 主な機能

### 1. 📋 SMIスコア診断 & 項目最適化
- **更年期レベルの可視化**: 婦人科等の臨床現場で用いられるSMI（簡略更年期指数）に基づき、現在のコンディションを数値化。
- **動的な項目選択ロジック**: SMIスコアの回答結果に応じ、その日のユーザーに最適なデイリーチェック項目を自動的に抽出・提示。

### 2. 📝 デイリー体調チェック（パーソナライズド）
- **対話型記録**: 体調・気分・疲労・出血量などをチャット形式で直感的に入力可能。
- **生理トラッキング**: `is_active` フラグを用いた、遡り入力や期間自動結合に対応した高度な生理期間管理。
- **気象連動**: Open-Meteo APIより気温・気圧を自動取得。
- **フェーズ判定**: 生理周期データに基づき「月経期」「卵胞期」などを自動判定し、ダッシュボードに反映。
- **AIアドバイス（準備中）**: 蓄積されたSMIスコアや周期データに基づき、Gemini 等のLLMを用いた個別最適化アドバイスの実装を予定。

### 3. 💬 コミュニティ & インサイト分析
- **Supabase連携**: 投稿、コメント、いいね機能を PostgreSQL で永続化。
- **日記機能**: 公開/非公開を選択可能な日記投稿機能。
- **質的データの蓄積**: 投稿内容の分析を通じて更年期世代の潜在的なインサイトおよびアンメットニーズを把握し、研究・サービス改善の質的データとして活用。

## 🧪 生理周期管理の技術的アプローチ
更年期の不規則な周期を正確に捉えるため、以下のロジックを独自実装しています。

* **`periods` テーブルの活用**: `is_active` カラムにより「現在進行中のフェーズ」を明示的に管理。
* **遡り入力・結合ロジック**: 過去日の記録時に既存レコードとの重複を確認し、`start` / `end` 日付を自動更新。
* **Single Source of Truth**: 生理状態の正解を `periods` テーブルに集約し、各画面の不整合を排除。

## 👥 技術相談・アドバイス歓迎（テストフェーズ）
現在、小規模なユーザーテストに向けた最終調整を行っています。特に以下の技術的課題について、知見をお持ちの方からのアドバイスやレビューを歓迎しています。

* **データベース（Supabase）の設計・最適化**:
    * 現在の `daily_checks`（体調記録）と `periods`（周期管理）のテーブル設計およびリレーションの妥当性。
    * データの整合性を保つための制約（Constraints）や効率的なクエリの書き方。
* **AI（Gemini API等）のセキュアな実装**:
    * クライアントサイドでの呼び出しから、Supabase Edge Functions 等を経由したセキュアなサーバーサイド実装への移行方法。
    * 更年期指数（SMI）や周期データを活用した、より精度の高いプロンプトエンジニアリングの提案。
* **アプリ化・モバイル展開（Expo / React Native）**:
    * 現在の React コードベースを活かした Expo への移行戦略。
    * モバイル特有のプッシュ通知やバックグラウンド処理の設計。
* **セキュリティ・安全性と運用保守**:
    * ヘルスケアデータを扱う上での、Supabase RLS（Row Level Security）の適切な設定。
    * フロントエンドおよびバックエンドにおけるエラーハンドリングとログ管理のベストプラクティス。
* **ウェアラブルデバイス連携の検討**:
    * Apple Watch などのスマートウォッチから取得できるバイタルデータ（心拍変動、睡眠の質、皮膚温など）を自動取得し、独自の体調記録・生理周期データと統合させるための最適なデータパイプラインの構築。
    * ウェアラブル由来の客観的指標と、SMI等の主観的指標を組み合わせた「更年期コンディション予測」の精度向上に向けたアルゴリズム設計。
